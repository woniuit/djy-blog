# vue3+elementplus表单封装

## 背景
![1](/19.png)

没有封装情况下

- 项目中表单过多的话代码会比较臃肿
- 修改某个点的话，需要每个文件去一一修改

封装了之后，只要更改配置文件就好

为了对比各种封装方法，下面会有演进的版本。

## 封装版本1

**子组件**

formData由父组件配置生成传递过来给子组件

```vue
<template>
  <div>
    <el-form label-width="100px" :model="formData" style="max-width: 460px">
      <template v-for="item in formData" :key="item">
          <el-form-item :label="item.label">
               <template v-if="item.type=='input'">
                   <el-input v-model="formData[`${item.field}`]" />
               </template>
          </el-form-item>
      </template>
    </el-form>
  </div>
</template>
<script setup lang="ts">
import { reactive, watch } from 'vue'
const props = defineProps({
  formData: {
    type: Array,
    default: () => []
  }
})
</script>

<style scoped lang="less"></style>

```

**父组件**

```vue
<template>
  <div>
    <forms :formData="formData.data" />
  </div>
</template>
<script setup lang="ts">
import { reactive, watch } from 'vue'
import Forms from '../../../../components/form'
const formData = reactive({
  data: [
    {
      label: '名字',
      type:"input",
      field:"name"
    },
    {
      label: '爱好',
      type:"input",
      field:"holp"
    }
  ]
})
components: {
  Forms
}
</script>

<style scoped lang="less"></style>
```

弊端：vue是单向数据流，这样做，子组件修改了由父组件传过来的对象。违反了vue的单向数据流原则。

## 封装版本2

利用双向绑定+计算属性computed

**子组件**

```vue
<template>
  <div>
    <el-form label-width="100px" style="max-width: 460px">
      <template v-for="item in formItems" :key="item">
        <el-form-item :label="item.label">
          <template v-if="item.type == 'input'">
            <el-input
              v-model="formData[`${item.field}`]"
            />
          </template>
        </el-form-item>
      </template>
    </el-form>
  </div>
</template>
<script setup lang="ts">
import { reactive, watch, computed } from 'vue'
const props = defineProps({
  modelValue: {
    type: Object,
    required: true
  },
  formItems: {
    type: Array,
    default: () => []
  }
})

const emit = defineEmits(['update:modelValue'])
const formData=computed({
    get:()=>props.modelValue,
    set:(newValue)=>{
        console.log('触发了吗') //没有触发
        emit('update:modelValue',newValue)
    }
})
</script>

<style scoped lang="less"></style>

```

**父组件**

```vue
<template>
  <div>
    <forms :formItems="formItems" v-model="formData" />
  </div>
</template>
<script setup lang="ts">
import { ref, watch } from 'vue'
import Forms from '../../../../components/form'
// import { FormConfig } from './config/index'
const FormConfig = {
  formItems: [
    {
      field: 'id',
      type: 'input',
      label: 'id',
      placeholder: '请输入id'
    },
    {
      field: 'name',
      type: 'input',
      label: '用户名',
      placeholder: '请输入用户名'
    },
    {
      field: 'realname',
      type: 'input',
      label: '真实姓名',
      placeholder: '请输入真实姓名'
    }
  ]
}
const formItems = FormConfig.formItems ?? []
// 双向绑定的属性应该是由配置文件的field来决定
// formData中的属性应该动态来决定
const formOriginData: any = {}
for (const item of formItems) {
  formOriginData[item.field] = ''
}
const formData = ref(formOriginData)
components: {
  Forms
}
</script>

<style scoped lang="less"></style>

```

弊端:用了computed还是违反了vue的单向数据流原则。

## 最终版本

modelValue+@update:modelValue+拷贝

**子组件**

```vue
<template>
  <div>
    <el-form label-width="100px" style="max-width: 460px">
      <template v-for="item in formItems" :key="item">
        <el-form-item :label="item.label">
          <template v-if="item.type == 'input'">
            <el-input
              :model-value="modelValue[`${item.field}`]"
              @update:modelValue="handleValueChange($event, item.field)"
            />
          </template>
        </el-form-item>
      </template>
    </el-form>
  </div>
</template>
<script setup lang="ts">
import { reactive, watch, computed } from 'vue'
const props = defineProps({
  modelValue: {
    type: Object,
    required: true
  },
  formItems: {
    type: Array,
    default: () => []
  }
})

const emit = defineEmits(['update:modelValue'])

const handleValueChange = (value: any, field: string) => {
      emit('update:modelValue', { ...props.modelValue, [field]: value })
    }
</script>

<style scoped lang="less"></style>

```
